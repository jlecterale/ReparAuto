## 1. Visão Geral

### O Que Resolve

Perguntas frequentes ("como publico um carro?", "como contacto o vendedor?", "é seguro?") consomem tempo de suporte e geram frustração quando não há resposta imediata. Um chatbot de atendimento, disponível em todas as páginas, responde a FAQs sobre a plataforma e pode orientar a busca ("procuro um carro até 2000 € no Porto"), usando o contexto da plataforma (regras, políticas, como funciona) para respostas relevantes. Alivia o suporte humano e mantém o utilizador dentro da plataforma.

### Benchmark Competitivo

| Plataforma | Chatbot de Atendimento |
|---|---|
| **Standvirtual** | Não |
| **AutoScout24** | Sim (básico, FAQ) |
| **Webmotors (BR)** | Sim (assistente de busca) |
| **Mercado Livre (BR)** | Sim (bot de ajuda + pós-venda) |
| **AutoTrader UK** | Sim |
| **OLX (PT/BR)** | Não |
| **RecarGarage (atual)** | Não |
| **RecarGarage (proposto)** | Sim (FAQ + orientação de busca) |

### Histórias de Utilizador

1. **Como visitante novo**, quero perguntar em linguagem natural como funciona a plataforma e obter resposta imediata, sem procurar em páginas de ajuda.
2. **Como comprador**, quero descrever o que procuro ("carro diesel até 5000 € perto de Braga") e ser orientado a usar a busca com esses filtros.
3. **Como vendedor**, quero tirar dúvidas rápidas sobre como publicar, editar ou destacar um anúncio sem abrir um pedido de suporte.

## 2. Especificação das Funcionalidades

### 2.1 Widget de Chatbot
Botão flutuante (bubble) presente em todas as páginas, no canto inferior direito (acima da BottomNav no mobile). Ao clicar, abre um painel de chat:
- **Mobile:** painel slide-up de largura total.
- **Desktop:** painel lateral (máx. 400px de largura, 600px de altura).

O chatbot:
- Responde em português (PT-PT), tom amigável e coloquial.
- Ajuda com: como publicar um carro, como pesquisar, políticas da plataforma, segurança, contactos.
- Quando perguntado sobre carros específicos, sugere usar a funcionalidade de busca com os filtros adequados.
- É claro sobre o que sabe e não sabe; nunca dá aconselhamento legal ou financeiro.
- Mensagem de saudação inicial: *"Olá! Sou o assistente do RecarGarage. Como posso ajudar?"*

### Fluxo de Utilizador
1. Utilizador clica na bubble → abre o painel com a saudação inicial.
2. Escreve uma pergunta → indicador de "a escrever" (pontos animados).
3. Cloud Function `chatbotResponse` recebe a mensagem + histórico (últimas ~10 mensagens) + system prompt com contexto da plataforma.
4. Resposta do LLM é apresentada como bolha do assistente.
5. Sessão persiste em `sessionStorage` (cliente) e, opcionalmente, na coleção `chatbotSessions` (Firestore) para continuidade.
6. Botão para limpar histórico.

### Requisitos de UI/UX
- Bubble redonda, ~56px, `shadow-lg`, `z-50`, margem de 16px, canto inferior direito.
- Pulse discreto na bubble quando há saudação disponível e o painel nunca foi aberto.
- Mensagens do assistente e do utilizador com estilos distintos (assistente à esquerda/neutro, utilizador à direita/destaque).
- Loading com pontos de digitação, não spinner.
- Renderização markdown-lite (negrito, listas) nas respostas.
- Reutilizar padrões de UI do chat existente (`src/components/chat/`) para consistência visual.
- Disclaimer discreto: *"Assistente automático — respostas geradas por IA."*
- Erros tratados com mensagem amigável em português.

### Coleção Firestore `chatbotSessions` (nova)
```typescript
interface ChatbotSession {
  id: string;
  uid?: string;              // null para utilizadores não autenticados
  mensagens: ChatbotMessage[];
  dataCriacao: Timestamp;
  dataUltimaMsg: Timestamp;
}

interface ChatbotMessage {
  role: 'user' | 'assistant';
  content: string;
  timestamp: Timestamp;
}
```

### Regras de Segurança Firestore
```
match /chatbotSessions/{sessionId} {
  allow read:   if isAuthenticated() && resource.data.uid == request.auth.uid;
  allow create: if true;   // permitir sessões anónimas
  allow update: if true;   // permitir adicionar mensagens
  allow delete: if false;  // não eliminar sessões
}
```

### Prompt de sistema (`CHATBOT_SYSTEM_PROMPT`)
Deve instruir o modelo a:
- Ser um assistente útil da plataforma RecarGarage; responder em PT-PT, tom amigável e conciso.
- Ajudar com: como publicar um carro, como pesquisar, políticas da plataforma, informação de contacto.
- Se perguntado sobre carros específicos, sugerir a funcionalidade de busca.
- Nunca dar aconselhamento legal ou financeiro.
- Ser claro sobre limitações ("não tenho acesso a...").

### Contexto e histórico
- Enviar apenas as últimas ~10 mensagens para limitar tokens.
- Usar Gemini 1.5 Flash (contexto suficiente para FAQ + histórico curto).
- Rate limiting recomendado: máx. ~20 mensagens por sessão/hora por utilizador anónimo.

## 5. Decisões de Arquitetura

### Decisão 1: Provider de IA
Ver Plano 4.1, Decisão 1 — **Google Gemini via Vertex AI** (decisão partilhada). Se, no futuro, a qualidade conversacional não for satisfatória, este componente pode migrar isoladamente para GPT-4o ou Claude sem afetar 4.1/4.3.

### Decisão 2: Persistência da sessão — só cliente vs. Firestore

| Opção | Prós | Contras |
|---|---|---|
| **sessionStorage apenas** | Simples; zero custo de leitura/escrita; privado por natureza | Perde histórico ao trocar de dispositivo; sem análise agregada |
| **sessionStorage + `chatbotSessions`** | Continuidade entre sessões (autenticados); permite análise de perguntas frequentes | Escritas Firestore; regras extra |

**Recomendação:** **sessionStorage como fonte primária** para v1 (rápido, privado), com escrita opcional em `chatbotSessions` para utilizadores autenticados (continuidade + insights de suporte). A coleção fica preparada mas o caminho crítico não depende dela.
