Importar Anúncios do Standvirtual

Um stand liga a sua conta (ou cola o URL do anúncio) e o RecarGarage importa os seus próprios anúncios do Standvirtual — marca, modelo, ficha técnica, preço, descrição e fotos — como rascunhos pendentes, prontos a rever e publicar. Ataca o maior travão à aquisição de stands: o custo de reescrever tudo à mão.

Nunca republica no Standvirtual
API oficial REST do Grupo OLX/Standvirtual — gratuita, mantida (v2.0.0)
3 vias API oficial · __NEXT_DATA__ por URL · integradores
Rascunho Importa como pendente; passa pela moderação normal
~5-7 d MVP (paste-URL); ~12-18 d total em 2 ondas

01. O que resolve & porquê ALTA prioridade

Um stand com 30-80 viaturas já as tem publicadas no Standvirtual (o portal dominante em PT), com fotos tratadas e ficha técnica preenchida. Pedir-lhe que reescreva tudo, viatura a viatura, no fluxo de 4 passos de app/anunciar é a barreira que o faz desistir antes de experimentar. Esta funcionalidade elimina esse atrito. O campo tipoConta: 'particular' | 'profissional' já existe em src/types/usuario.ts — a ferramenta é exclusiva de contas profissionais, alinhada com os planos 07 e 17.

📈

Aquisição de oferta

Importar um stand médio adiciona 30-80 anúncios de qualidade num clique — o inverso de crescer anúncio a anúncio.

✨

Diferenciador

Nenhum portal PT oferece isto como funcionalidade self-serve nativa; só uma ferramenta de gestão de stands (Auto Connect) o faz, como plugin pago.

🛡️

Risco técnico baixo

Existe API oficial documentada que devolve exatamente os dados necessários. Não é um projeto de scraping frágil.

03. Via A — API oficial (recomendada)

API REST de primeira parte, documentada e ativamente mantida (a referência api_data.json foi regenerada em 2026-07-03, versão 2.0.0). Mesma base FixeAds/OLX do Otomoto. Verificada por probes live: GET /account/adverts devolveu 401 invalid_token — existe, só falta credencial.

Base & docs

https://www.standvirtual.com/api/open
Ref: /api/doc/ · Registo: developer.olxgroup.com/register?market=motors · api@standvirtual.com

Custo & pré-requisito

A API é gratuita (o stand paga as taxas normais do portal). Aprovação discricionária; requer conta profissional do lado do stand.

Autenticação — OAuth2 (POST /oauth/token/)

  • password grant — client_id+client_secret (RecarGarage) + username/password (stand). Age em nome do stand — o nosso caso de uso.
  • partner grant — partner_code/partner_secret. Sem passwords de stands; alvo desejável a médio prazo.
  • Token de 12 h (expires_in: 43200) + refresh_token; header User-Agent obrigatório.

Endpoints de leitura (o caminho de importação)

GET /account/advertsLista todos os anúncios do stand, paginado (total_pages, is_last_page).
GET /account/adverts/:idAnúncio completo: params (make, model, fuel_type, mileage, power, price…) + photos em CDN (1080×720).
GET /account/adverts/:id/statsEstatísticas de visualizações.
GET /imageCollections/:idColeções de imagens.
GET /categories · /makes · /modelsTaxonomia para traduzir slugs → enums do RecarGarage; /regions·/cities para localização.

Via B (fallback leve): o site é Next.js SSR — cada anúncio traz o objeto completo em __NEXT_DATA__.props.pageProps.advert (incl. parametersDict com VIN e ~60 flags de equipamento). Serve para «cola o URL → formulário pré-preenchido», sem credenciais. Telefone vem tokenizado; anti-bot DataDome é passivo em baixo volume. Manter iniciado pelo utilizador, um anúncio de cada vez.

05. Arquitetura & custódia de credenciais

O fetch tem de ser server-side (CORS, custódia do client_secret, User-Agent obrigatório). O projeto ainda não tem app/api/ — esta funcionalidade introduz as primeiras Route Handlers do App Router.

app/api/import/standvirtual/
 ├── connect/route.ts   # POST — password grant → refresh_token cifrado
 ├── adverts/route.ts   # GET — lista paginada dos anúncios do stand
 ├── advert/[id]/route.ts # GET — 1 anúncio já mapeado (preview)
 └── import/route.ts    # POST — cria cars (pendente) + copia fotos
src/lib/importers/ standvirtual.client.ts · .nextdata.ts · .map.ts (TDD) · photos.ts
1

Nunca guardar a password

Trocar por token → guardar só o refresh_token cifrado (KMS/segredo) num doc server-only integrations/{uid}, descartar a password. MVP pode usar sessão efémera (não persiste nada).

2

Regras fechadas ao cliente

match /integrations/{uid} { allow read, write: if false; } — só acessível via Admin SDK. Segredos STANDVIRTUAL_CLIENT_ID/SECRET nunca em NEXT_PUBLIC_.

3

Fotos re-hospedadas

Descarregar do CDN OLX e re-hospedar no Firebase Storage (o next/image só permite firebasestorage/Google em remotePatterns). Reutiliza addCarro e uploadFileToStorage — sem duplicar lógica.

06. Legal & confiança

A distinção-chave: um stand a importar os seus próprios anúncios, por sua iniciativa e consentimento, é material e eticamente distinto de raspar dados de concorrentes. É esse o padrão a construir — nunca oferecer «importa os anúncios do stand X».

  • ToS Standvirtual (Prof., Mai 2025): restringe acesso automatizado e extração da base de dados. Rever o texto integral antes de construir.
  • Ryanair v PR Aviation (C-30/14): os ToS podem proibir scraping por contrato → manter iniciado pelo stand, autenticado, baixo volume.
  • RGPD: telefone é dado pessoal (tokenizado de propósito). Não reconstruir — usar o número do perfil RecarGarage.
  • Direitos de imagem (maior risco): CarGurus v. Trader — fotos têm copyright do autor. Importar só com atestação do stand; guardar trilho de auditoria (URL + consentimento + timestamp); oferecer «importar só dados».

🏁 Benchmark

  • Auto Connect (PT) — já importa diretamente do Standvirtual. Precedente direto.
  • MyStand · 4Stand · OfficeGest · BETACODE — hub-and-spoke: empurram stock para os portais via API.
  • OLX Brasil — Autoupload API pública (JSON, OAuth2, fotos obrigatórias máx. 20).
  • mobile.de / AutoScout24 / CarGurus — feeds CSV/XML padrão; ingestão tolerante.
  • O próprio Standvirtual — prefill por matrícula/VIN + IA de criação em massa.

Aviso (Facebook): descontinuou os anúncios grátis de viaturas — canais de importação grátis acabam monetizados ou fechados.

⚠️ Riscos & Casos de Borda

  • Duplicados: detetar por origemId — atualizar ou saltar (dor nº 1 dos integradores).
  • Valor não mapeável: não bloquear — importar vazio e sinalizar revisão.
  • Foto falha download: importar na mesma, marcar em falta, permitir re-upload.
  • Token expira a meio: refrescar e continuar; se falhar, parar com estado parcial claro.
  • DataDome desafia (Via B): degradar com graça → empurrar para a Via A.
  • Só active por defeito: não importar carros já vendidos sem opt-in.

🔓 Decisões em Aberto

  • Credenciais API — registar no Grupo OLX (grátis, discricionária). Bloqueia a Via A; contornável começando pela Via B.
  • password vs partner grant — MVP com sessão efémera; migrar para partner (sem passwords).
  • Fotos: re-hospedar (recomendado) vs URLs externos.
  • ToS Mai 2025 — validação jurídica antes do lançamento.

🚀 Execução em 2 Ondas

  1. Onda 1 (~5-7 d): paste-URL de 1 anúncio (Via B) + campos de origem, TDD-first, sem credenciais. Valor imediato.
  2. Onda 2 (~7-11 d): ligação de conta + import em lote via API oficial (Via A), quando as credenciais forem aprovadas.
  3. Paralelo: Via C (parceria com integradores) como jogada de negócio para stock multi-stand.
