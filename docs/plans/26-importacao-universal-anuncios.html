🔌 Investigação — Expansão do Importador de Anúncios (Plano 24)

Importação Universal de Anúncios (Qualquer URL)

Depois do Standvirtual, o importador passa a aceitar qualquer URL de anúncio — OLX Portugal, OLX Brasil, Webmotors, Mercado Livre, CustoJusto e qualquer outro portal que o vendedor use. Usa uma arquitetura de extração em 4 camadas (parser nativo → schema.org → LLM → revisão manual) para extrair marca, modelo, ano, preço, km e fotos, pré-preencher o formulário e criar o anúncio como pendente no RecarGarage.

Qualquer URL · Qualquer Plataforma
20+ plataformas Cobertura de Portugal e Brasil combinados (OLX, Webmotors, Mercado Livre, CustoJusto, iCarros, Mobiauto…)
4 camadas Extração progressiva: parser dedicado → JSON-LD schema.org → LLM → preenchimento manual
Mesma base Reutiliza Rate Limiter, re-hospedagem de fotos e fluxo de moderação do Plano 24
~8-11 d MVP (OLX PT + schema.org + LLM); ~16-22 d cobertura completa

01. O que resolve & porquê ALTA prioridade

O Plano 24 atacou o Standvirtual (portal dominante em Portugal), mas o mercado é fragmentado: stands e particulares anunciam em 3-5 plataformas diferentes. O OLX Portugal sozinho tem +15.000 anúncios de carros; o OLX Brasil e Webmotors juntos somam centenas de milhares. Um stand que usa OLX + Standvirtual + CustoJusto ainda teria de reescrever 2/3 do inventário à mão. Esta funcionalidade elimina essa barreira para todas as plataformas. O campo tipoConta já existe em src/types/usuario.ts — a ferramenta é exclusiva de contas verificadas (profissionais com documentação validada ou particulares com email verificado).

📈

Aquisição de oferta

Cada plataforma adicional coberta desbloqueia um novo segmento de vendedores que já têm anúncios prontos noutro lado.

🔬

Estratégia em camadas

Não tentamos um parser universal frágil. A arquitetura de 4 camadas dá o melhor resultado possível para cada URL, com degradação graciosa.

🛡️

Risco distribuído

Cada camada é independente. Se uma plataforma muda o HTML, só o parser dedicado dela quebra — as outras camadas continuam a funcionar como fallback.

02. As quatro camadas de extração

Recomendação: construir as 4 camadas em paralelo, priorizando os parsers dedicados de maior volume (OLX PT → OLX BR → Webmotors).

Camada Abordagem Cobertura Precisão Fragilidade Custo
1 — Parser dedicado Regex/JSON do HTML da plataforma (como o __NEXT_DATA__ do Standvirtual) 1 plataforma Alta (95%+) Média (muda HTML) Baixo (dev)
2 — schema.org JSON-LD Vehicle ou Product + OpenGraph ~60% dos portais Média (70-85%) Baixa (standard W3C) Zero marginal
3 — LLM Enviar HTML limpo para Gemini/Claude extrair JSON estruturado Qualquer URL Boa (80-90% c/ prompt bom) Muito baixa (modelo estável) ~$0.001-0.01/req
4 — Revisão manual Campos não mapeados ficam vazios; utilizador preenche no wizard 100% N/A (humano) Nenhuma Zero adicional

Fluxo de extração: O utilizador cola um URL → detectPlatform(url) identifica a plataforma → Tenta Camada 1 (se plataforma conhecida) → Tenta Camada 2 (JSON-LD) → Tenta Camada 3 (LLM, se dados insuficientes) → Camada 4 (wizard pré-preenchido com o que foi extraído). O utilizador vê uma barra de progresso com checkmarks por camada.

1

Parser genérico em pipeline

detectPlatform identifica a plataforma → se conhecida, aplica o parser dedicado → extrai NormalizedAdvert → se não, tenta schema.org → extrai o que conseguir → se dados < 50% dos campos obrigatórios, escala para LLM.

2

Schema.org como denominador comum

60%+ dos portais de classificados usam JSON-LD com tipos Vehicle ou Product. Extraímos name, description, offers.price, image, brand.name, model, vehicleModelDate, mileageFromOdometer, fuelType, vehicleTransmission. Muito mais leve e confiável que parsing de HTML.

3

LLM como fallback universal

A Cloud Function (ou Route Handler) envia o HTML limpo (sem scripts, estilos, comentários) para Gemini 2.0 Flash (~$0.00002/1K tokens) com um prompt que pede JSON estruturado. O output é validado contra o schema CarroFormData antes de ser aceite. O HTML é truncado a 64K tokens. Usar o modelo mais barato possível (Gemini Flash ou Claude Haiku).

4

Fotos: whitelist dinâmica de CDNs

Em vez de um SSRF guard por plataforma (isAllowedPhotoUrl atual), manter uma whitelist de CDNs conhecidos (olxcdn.com, webmotors.com.br, http2.mlstatic.com, etc.) mais um validador genérico que só permite URLs de imagem com extensão conhecida e sem portas. Cada plataforma regista os seus domínios de CDN.

5

Reutilização máxima do Plano 24

O Rate Limiter, a re-hospedagem de fotos, o fluxo de preview→draft→wizard, o badge "Importado", a deteção de duplicados por origemId, o attestOwnership e as notificações de admin — tudo é reutilizado. A interface NormalizedAdvert é partilhada entre todos os parsers.

05. Interface NormalizedAdvert (contrato universal)

Todos os parsers (dedicados, schema.org e LLM) produzem esta interface. O map.ts genérico converte para CarroFormData.

interface NormalizedAdvert {
  // Identificação da fonte
  platform: string;            // 'olx_pt' | 'webmotors' | 'mercadolivre' | ...
  sourceUrl: string;
  externalId: string;          // ID único na plataforma de origem
  extractedAt: string;         // ISO timestamp
  extractionTier: 'parser' | 'schemaorg' | 'llm';

  // Dados do veículo (todos opcionais — extraímos o que conseguimos)
  title?: string;
  brand?: string;
  model?: string;
  version?: string;
  year?: number;
  registrationMonth?: number;
  mileage?: number;
  price?: { amount: number; currency: string };
  fuelType?: string;
  transmission?: string;
  bodyType?: string;
  color?: string;
  doors?: number;
  seats?: number;
  enginePower?: number;
  engineDisplacement?: number;
  traction?: string;
  condition?: string;
  description?: string;
  features?: string[];
  location?: { city?: string; region?: string; district?: string };
  origin?: string;
  previousOwners?: number;
  warrantyMonths?: number;
  acceptsFinancing?: boolean;
  acceptsExchange?: boolean;
  photos: { url: string; order: number }[];

  // Metadados de extração
  unmappedFields: string[];    // Campos que existem na origem mas não mapeamos
  extractionConfidence: number; // 0-1 (parser dedicado ~0.95, schema.org ~0.75, LLM ~0.85)
}

O map.ts genérico recebe um NormalizedAdvert e devolve CarroFormData + unmappedFields. Os parsers de plataforma produzem NormalizedAdvert; as camadas 2 e 3 também. Isto significa que qualquer nova plataforma é só um novo produtor de NormalizedAdvert — o resto do pipeline não muda.

06. Legal & confiança

A expansão para "qualquer URL" amplifica os riscos legais do Plano 24. O framework ético mantém-se: a ferramenta é para importar os seus próprios anúncios, não para raspar concorrentes.

  • Atestação de propriedade reforçada: O checkbox atual ("Declaro que sou o proprietário legítimo") passa a exigir dupla confirmação para plataformas onde não temos relação contratual. Texto explícito: "Compreendo que importar anúncios de terceiros viola os Termos de Uso do portal de origem e pode resultar na remoção dos meus anúncios e suspensão da conta."
  • ToS por plataforma: OLX Group (Standvirtual + OLX PT + OLX Brasil) partilha os ToS do Grupo OLX — restringe acesso automatizado. Webmotors e Mercado Livre têm políticas semelhantes. A Camada 1 (parser dedicado) é a mais exposta; as Camadas 2-3 (schema.org + LLM) extraem dados que o próprio portal expõe como metadados públicos para motores de busca (Google, Bing) — o que reduz significativamente o risco.
  • RGPD / LGPD: Telefone nunca é extraído (vem do perfil RecarGarage). Dados pessoais do vendedor (nome, localização exata) são extraídos apenas se já forem públicos no anúncio original. O utilizador é o responsável pelo tratamento — o RecarGarage é um subcontratante que processa dados por instrução do utilizador.
  • Direitos de imagem (maior risco): Tal como no Plano 24, fotos são re-hospedadas com atestação do utilizador. O trilho de auditoria (origemUrl + importadoEm + criadorUid) fica gravado em Firestore. Oferecer opção "Importar sem fotos" para utilizadores cautelosos.
  • Robots.txt e rate limiting: O fetcher respeita robots.txt por domínio (cache de 1 h). O rate limit é por utilizador (20/min, 120/h) independentemente da plataforma — a mesma janela deslizante do Plano 24.

🏁 Benchmark

  • Auto Connect (PT) — importa do Standvirtual; não cobre OLX nem CustoJusto.
  • 4Stand / MyStand (PT) — DMS que empurram stock para portais (sentido contrário: do stand para os portais).
  • OLX Brasil Autoupload API — API pública REST/JSON para publicar anúncios em lote. Sentido contrário ao nosso.
  • ChatGPT / Claude — utilizadores já copiam o texto do anúncio e pedem à IA para "extrair os dados". A Camada 3 automatiza isto.
  • Zap+ / DataZap (BR) — agregadores que aspiram anúncios de múltiplos portais (sentido contrário: leitura, não escrita). Têm parsers para Webmotors, OLX, iCarros — prova de que é tecnicamente viável.

Importante: Nenhum portal oferece "importe os seus anúncios de qualquer site" como funcionalidade self-service. Existe um espaço vazio para ocupar.

⚠️ Riscos & Casos de Borda

  • Anti-bot (Cloudflare/DataDome): Plataformas como Webmotors e Mercado Livre usam proteção agressiva. O parser dedicado pode ser bloqueado. Mitigação: Camada 2 (schema.org) extrai dados sem precisar de bypass; Camada 3 (LLM) funciona com HTML parcial. Se tudo falhar, o utilizador ainda pode copiar/colar manualmente.
  • Mudança de HTML: O parser de uma plataforma pode partir de um dia para o outro. Mitigação: testes de snapshot com fixtures reais; monitorização (quando um parser falha recorrentemente, alerta o time); fallback automático para Camada 2/3.
  • Custos de LLM: Se 10.000 utilizadores importarem 5 anúncios cada via LLM, são ~$50-250/mês. Mitigação: rate limit generoso (20/dia por utilizador para LLM); cache de respostas do LLM para URLs idênticas (1 h).
  • Qualidade variável: Anúncios mal escritos (ex.: "vendo carro bom estado tratar com Zé") extraem poucos dados. Mitigação: a Camada 4 (revisão manual) é sempre o fallback final. A UI mostra claramente o que foi extraído vs. o que precisa de ser preenchido.
  • Duplicados cross-plataforma: Um carro anunciado no OLX e no Standvirtual gera 2 anúncios distintos no RecarGarage. Mitigação: deteção de duplicados por similaridade de título + preço + km + localização (opcional, v2). Na v1, o utilizador é avisado se o origemId já existe.
  • Fotos de baixa qualidade ou marca d'água: Alguns portais aplicam marcas d'água. Mitigação: não tentamos remover (risco legal); o utilizador pode substituir as fotos no wizard.

🔓 Decisões em Aberto

  • LLM provider: Gemini 2.0 Flash (mais barato, já planeado no Plano 4.1) vs Claude Haiku (melhor em extração estruturada). Decidir com teste A/B com 50 anúncios reais.
  • Ordem das plataformas: OLX PT primeiro (mesmo grupo do Standvirtual, estrutura similar) ou OLX Brasil (maior volume)? Critério: proximidade técnica vs. impacto de mercado.
  • Parser dedicado para OLX PT/BR: Usar a API Autoupload do OLX Brasil (se aprovados como parceiros) ou parsing do HTML? A API é o caminho correto mas requer aprovação — mesma situação da Via A do Standvirtual.
  • Custo do LLM por utilizador: Absorver o custo (investimento em aquisição) ou limitar a X importações/mês no plano gratuito? Sugestão: ilimitado para profissionais premium; 5/mês para particulares.
  • Cache de extrações: Fazer cache dos resultados de schema.org/LLM por URL (1 h)? Reduz custos e latência mas pode servir dados desatualizados. Sugestão: cache de 1 h com invalidação se o utilizador pedir "reatualizar".

🚀 Execução em 3 Ondas

  1. Onda 1 (~8-11 d): Motor genérico + OLX Portugal + schema.org + LLM. Cobre ~80% dos URLs que um utilizador colaria. Infraestrutura core (core/), extratores, LLM, detectPlatform, parser OLX PT, unificação do NormalizedAdvert. Sem batch mode — só 1 anúncio de cada vez (preview → wizard).
  2. Onda 2 (~5-7 d): OLX Brasil + Webmotors + Mercado Livre. Parsers dedicados para os 3 maiores portais brasileiros. Batch mode generalizado (reutiliza a lógica do BatchImportPanel). Whitelist de CDNs brasileiros.
  3. Onda 3 (~3-4 d): CustoJusto + AutoScout24 + iCarros + Mobiauto. Cobertura de cauda longa. Melhorias de UX (badge de confiança, cache de extrações, preview de qualidade).
