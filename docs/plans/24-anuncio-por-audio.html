Anúncio por Áudio (Web + Mobile)

Gravar ou enviar ficheiro de áudio e extrair automaticamente campos de formulário estruturados com IA Gemini (WAV encoder no cliente, Interactions API no backend).

Implementado & Entregue
Gemini 3.5 Interactions API com Flash
WAV Client Resample de 16 kHz Mono PCM
< 1 min Tempo médio de criação de anúncio
12 MB Limite de áudio (MP3/WAV/AAC/etc)

01. Contexto e o que resolve

Criar um anúncio completo exige preencher cerca de 15 campos num ecrã pequeno. Muitos vendedores sabem descrever o veículo de cor ("é um Renault Clio de 2012, gasolina, caixa manual, 180 mil km, cinzento, 5 portas, quero 4500 euros…") mas desistem a meio do formulário devido ao atrito.

Este plano adiciona um assistente inteligente de áudio à criação de anúncios (veículo e peça) na Web e no Mobile, permitindo ao utilizador preencher todos os dados apenas falando, restando apenas o carregamento das fotografias.

02. Porquê Gemini direto e Multimodal?

Comparação das abordagens tecnológicas possíveis para preenchimento de formulários por voz.

Opção Prós Contras
Web Speech API (Browser) Grátis, sem consumo de backend. Sem suporte no Firefox/WebViews, qualidade irregular. Requer LLM adicional para extração.
STT Dedicado (Whisper/Google STT) Transcrição robusta. Apenas transcreve; exige 2 chamadas de rede (STT + LLM) duplicando integrações e custo.
Gemini Multimodal (Interactions API) ✅ Uma única chamada: envia áudio e recebe JSON estruturado, entende PT-PT/PT-BR nativamente, muito rápido. Exige GEMINI_API_KEY do lado do servidor.

Conversão de áudio: O Gemini oficialmente suporta WAV, MP3, AIFF, AAC, OGG e FLAC. Visto que o MediaRecorder dos navegadores grava em webm/opus, a Web converte deterministicamente o áudio no cliente para WAV (16 kHz mono PCM) usando a Web Audio API antes do envio. No mobile, o expo-audio grava em .m4a (AAC), suportado nativamente.

03. Âmbito / Arquitetura

Servidor / API

  • app/api/listing-from-audio/route.ts: Rota autenticada via Firebase ID-token para evitar abusos. Chama a Interactions API da Google usando o SDK @google/genai.
  • src/lib/audioListing.ts: Lógica de domínio TDD contendo esquemas JSON estruturados e sanitização pós-extração (validação de enums de combustível, caixa, limites de portas/quilometragem/preço e mapeamento concelho/distrito).
  • Utiliza a variável de ambiente GEMINI_API_KEY. Se não estiver configurada, a API responde com 503 e a UI oculta o botão elegantemente.

Web / Cliente

  • src/lib/audioWav.ts: Encoder WAV puro em JS (downmix mono + resample para 16 kHz).
  • src/components/anunciar/AudioAdAssistant.tsx: Componente interativo que permite gravar áudio (limite de 3 min) ou selecionar arquivo físico do computador, com a11y integrada.
  • Integrado no topo do StepDados (formulário de carros) e no PecaForm (formulário de peças).

Mobile (React Native / Expo)

  • Utiliza o pacote expo-audio para gravação local e expo-document-picker para selecionar ficheiro.
  • mobile/src/components/anunciar/AudioAdAssistant.tsx: Componente móvel estilizado via NativeWind.
  • ⚠️ Atenção: Por requerer novos pacotes nativos e permissões do sistema (microfone), este plano **exige uma nova build nativa (.apk / .ipa)** para os testadores de telemóvel.

04. Benchmark Competitivo

  • OLX Brasil (2025): Criação de anúncios preenchidos por IA a partir de **fotografias**, mas o vendedor ainda precisa escrever a descrição e preencher quilometragem/cor à mão.
  • ZAP Imóveis: Permite fazer a *busca* de imóveis por comandos de voz, mas não a criação de anúncios.
  • Stands (Standvirtual, Webmotors, etc.): Utilizam pré-preenchimento via base de matrículas/placas ou introdução exata de marca-modelo; nenhum oferece preenchimento contextual por voz.

⚠️ Casos de Borda & Regras

  • Sem áudio / Ruído: O Gemini retorna JSON vazio e a UI avisa "Não conseguimos extrair informações do áudio. Tente falar novamente".
  • Valores absurdos: Clamps de sanidade rejeitam carros de ano < 1900 ou quilometragem superior a 2.000.000 km.
  • Sobreposição de dados: Se o utilizador já digitou dados nos campos, o assistente preenche **apenas os campos vazios**, respeitando o input manual do utilizador.
  • Falta de Permissões: Caso a permissão do microfone seja negada, o assistente oculta a gravação imediata e mostra apenas o botão de upload de arquivo de áudio.
