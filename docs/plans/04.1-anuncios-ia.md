# Plano 4.1 — Anúncios com IA

**Prioridade:** MÉDIA
**Estimativa:** 14-16 dias de desenvolvimento
**Impacto Principal:** Qualidade dos anúncios, redução de esforço do vendedor, diferenciação

> Este plano é uma das três partes do antigo Plano 4 (*Funcionalidades com IA*), dividido em **4.1 Anúncios com IA**, **4.2 Chatbot de Atendimento** e **4.3 Detecção de Fraude**. As três partes partilham a mesma decisão de provider de IA (Google Gemini via Vertex AI) e o mesmo setup de Firebase Cloud Functions (ver secção *Dependência Partilhada*).

## 1. Visão Geral

### O Que Resolve

A qualidade dos anúncios é um dos maiores desafios de qualquer marketplace. Vendedores não sabem escrever descrições atrativas, não sabem que preço pedir, e fotos de veículos com danos não são facilmente interpretadas por compradores. Este plano usa IA para elevar a qualidade do próprio anúncio em três frentes:

1. **Geração automática de descrição** a partir dos dados já preenchidos.
2. **Sugestão inteligente de preço** com explicação do raciocínio.
3. **Classificação visual de danos** nas fotos, apresentada como overlay.

Todas atuam no momento da criação/consulta do anúncio, reduzem o esforço do vendedor e ajudam o comprador a decidir com mais informação.

|---|---|---|---|
| **Standvirtual** | Não | Não | Não |
| **AutoScout24** | Não | Parcial | Não |
| **Webmotors (BR)** | Não | Parcial (tabela FIPE) | Não |
| **Carvana (EUA)** | Sim | Sim | Parcial |
| **AutoTrader UK** | Não | Sim | Não |

Nenhum concorrente direto em Portugal ou no Brasil oferece geração de descrição por IA — é um diferenciador de marketing forte ("a IA escreve o anúncio por si").

### Histórias de Utilizador

1. **Como vendedor que não sabe escrever descrições**, quero que a plataforma gere automaticamente uma descrição profissional do meu carro a partir dos dados que já preenchi, para que o meu anúncio seja mais atrativo e completo.
2. **Como vendedor inseguro sobre o preço**, quero uma sugestão de preço explicada (não apenas um número), para publicar com confiança e reduzir o tempo até à venda.
3. **Como comprador a ver fotos de um carro com danos**, quero que a plataforma me indique automaticamente onde estão os danos visíveis, para avaliar a extensão dos problemas antes de contactar o vendedor.

## 2. Especificação das Funcionalidades

### 2.1 Geração Automática de Descrição
No formulário de criação de anúncio (`/anunciar`), depois de o vendedor preencher os dados básicos (marca, modelo, ano, km, combustível, estado, manutenções) e carregar fotos, um botão **"Gerar Descrição com IA"** envia estes dados a um LLM que devolve um texto descritivo profissional em português. O vendedor pode editar o resultado antes de publicar. Marca-se `descricaoGeradaIA: true` no documento.

### 2.2 Sugestão Inteligente de Preço
Extensão da Inteligência de Preços (Plano 02) com camada de IA. Além da mediana estatística, o LLM analisa os dados do veículo (incluindo manutenções necessárias e estado) e sugere um preço mais preciso, **explicando o raciocínio**: *"Sugerimos X € porque veículos semelhantes custam Y €, mas descontámos Z € pelas manutenções necessárias."* Distingue-se visualmente da sugestão puramente estatística (badge "Assistido por IA").

### 2.3 Classificação Visual de Danos
O utilizador faz upload de fotos do veículo. A IA de visão (Gemini Vision) analisa cada foto e identifica áreas com danos visíveis (amassados, riscos, ferrugem, peças em falta, vidro partido). O resultado é apresentado como overlay opcional na foto, com bounding boxes e labels. Os resultados são cacheados no documento do carro para evitar reanálise.

### Fluxos de Utilizador

**Fluxo de Geração de Descrição:**
1. Vendedor está no passo de dados do formulário `/anunciar`.
2. Campos obrigatórios preenchidos (marca, modelo, ano).
3. Botão "Gerar com IA" fica ativo → spinner + chamada à Cloud Function.
4. Cloud Function envia prompt ao LLM com os dados do veículo.
5. Resposta é inserida no textarea de descrição (efeito typewriter opcional).
6. Vendedor edita livremente antes de submeter.
7. Indicador discreto: "Descrição gerada com auxílio de IA".

**Fluxo de Sugestão de Preço:**
1. No passo de preço, botão "Obter Sugestão de Preço com IA".
2. Envia dados do veículo + mediana de mercado (do Plano 02, se disponível).
3. Cloud Function devolve `{ precoSugerido, precoMinimo, precoMaximo, explicacao }`.
4. Card mostra preço sugerido (grande), intervalo (min-max) e explicação.
5. Botão "Usar este preço" preenche o campo de preço.

**Fluxo de Classificação de Danos:**
1. Na página de detalhes de um carro com `estadoVeiculo === 'manutencao'` (ou automático no upload para o vendedor).
2. Botão "Analisar Danos com IA" → fotos enviadas à Cloud Function de visão.
3. Resultado: JSON com coordenadas de danos por foto (cacheado no documento).
4. UI sobrepõe bounding boxes semi-transparentes com labels ("Amassado", "Risco", "Ferrugem").
5. Toggle para mostrar/esconder overlay.

### Requisitos de UI/UX
- Botões de IA com ícone de sparkle/varinha e background em gradiente (distintos dos botões normais) para sinalizar "AI-powered".
- Loading de IA: pontos animados ou shimmer, não spinner simples.
- Descrição gerada pode aparecer com efeito de digitação (typewriter) para reforçar a experiência.
- Overlay de danos opcional, não obstruindo a visualização normal; label posicionado fora da caixa; cores por tipo de dano.
- Sugestão de preço com IA diferencia-se da estatística (badge "Assistido por IA").
- Disclaimer claro em todas as funcionalidades: *"Resultado gerado por IA — verifique antes de utilizar."*
- Erros de API tratados com mensagem amigável em português; nunca expor erro cru.

## 3. Implementação Técnica

### Dependência Partilhada — Setup de Cloud Functions
Este plano (e os planos 4.2 e 4.3) exige **Firebase Cloud Functions (2ª geração, Node.js 18+)**. Se a pasta `functions/` ainda não existir, inicializar uma vez:
- `firebase init functions` (TypeScript, 2nd gen) → cria `functions/`.
- `cd functions && npm install @google-cloud/vertexai`.
- Todas as funções são **HTTPS Callable** (não HTTP triggers), exceto a de deteção de fraude (Plano 4.3, que é trigger `onDocumentCreated`).
- Provider de IA: **Google Gemini via Vertex AI** (integrado no Firebase). Ver Decisão 1.

### Avaliação de Valor

| Dimensão | Impacto | Justificação |
|---|---|---|
| **Aquisição** | Alto | "IA que escreve o anúncio por si" é diferenciador único no mercado PT/BR |
| **Qualidade** | Muito Alto | Descrições mais completas e profissionais elevam a qualidade média dos anúncios |
| **Retenção** | Médio | Reduz o atrito de publicar; sugestão de preço acelera a venda |
| **Receita** | Médio | Funcionalidades podem ser monetizadas (geração premium, análise de danos por consulta) |

**Posição na matriz:** COMPLEMENTAR tendendo a ESTRATÉGICO. A geração de descrição tem a melhor relação esforço/valor e deve ser feita primeiro; a classificação de danos é a mais complexa e pode ser a última desta parte.

## 5. Decisões de Arquitetura

### Decisão 1: Provider de IA (partilhada com 4.2 e 4.3)

| Opção | Prós | Contras |
|---|---|---|
| **Google Gemini (Vertex AI / Firebase)** | Integração nativa Firebase; Gemini 1.5 Flash económico; multimodal (texto + visão); billing integrado; português nativo | Menos potente que GPT-4o/Claude em raciocínio complexo |
| **OpenAI GPT-4o** | Muito capaz; excelente português; visão avançada | Sem integração Firebase; custo 2-3x; API key externa |
| **Anthropic Claude API** | Excelente português; contexto longo | Sem integração Firebase; custo maior; visão menos robusta |

**Recomendação:** **Google Gemini via Vertex AI**. A integração nativa com Firebase é determinante — elimina a gestão de API keys externas, simplifica o billing e as Cloud Functions chamam Vertex AI diretamente. O Gemini Vision (multimodal) cobre a análise de danos sem um modelo separado.

### Decisão 2: Classificação de Danos — API genérica vs. modelo próprio

| Opção | Prós | Contras |
|---|---|---|
| **Gemini Vision (prompt engineering)** | Sem treino; flexível; rápido de implementar; custo baixo por imagem | Menos preciso; bounding boxes aproximadas; depende do prompt |
| **Modelo próprio (YOLOv8 fine-tuned)** | Muito preciso; bboxes precisas | Requer dataset anotado; infra de ML; custo inicial alto |

**Recomendação:** **Gemini Vision com prompt engineering** para a v1. O custo/esforço de treinar um modelo próprio é desproporcional à fase atual. Resultados cacheados no documento do carro. Evoluir para modelo *fine-tuned* só se o volume e a qualidade o justificarem.
